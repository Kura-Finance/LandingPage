# Security Policy

## Supported versions

Security fixes are applied to the latest release on the `main` branch.

| Version | Supported |
| --- | --- |
| Latest on `main` | Yes |
| Older tags | No |

## Reporting a vulnerability

If you discover a security issue in this repository, please report it responsibly.

**Do not** open a public GitHub issue for security vulnerabilities.

Instead, email **support@kura-finance.com** with:

- A description of the vulnerability
- Steps to reproduce
- Potential impact
- Any suggested fix (optional)

We aim to acknowledge reports within **3 business days** and will keep you updated on remediation progress.

## Scope

This policy covers the **LandingPage** repository (marketing site source code). It does not cover:

- The Kura mobile or web app
- Third-party partner services (Bridge, MoonPay, Dinari, Gnosis Pay, etc.)
- Infrastructure or deployment credentials outside this repo

## Safe harbor

We appreciate good-faith security research. We will not pursue legal action against researchers who follow this policy and avoid privacy violations, data destruction, or service disruption.
